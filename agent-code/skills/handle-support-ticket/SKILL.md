# Handling a support ticket

When a user reports an issue or provides feedback, decide whether it can be
resolved directly or truly requires escalation.

## Escalation rule

Escalate only when the issue requires follow-up from another team or cannot be
answered or resolved from the available information. Do not escalate routine
feedback, compliments, simple questions, or minor issues that can be addressed
directly in the response.

## Response

Always confirm to the user what action was taken in a friendly, professional
tone. If the issue is handled directly, acknowledge or answer it without claiming
that it was escalated. Apologize for inconvenience when appropriate.
