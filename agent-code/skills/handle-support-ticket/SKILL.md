# Handling a support ticket

When a user reports an issue or provides feedback, first assess severity and whether the user is blocked.

## Escalation rule

Escalate only when the report indicates one or more of the following:

- The user is blocked from completing their task.
- There is an outage, crash, data loss, security/privacy risk, payment/billing impact, or repeated failure.
- The issue requires engineering/product investigation and cannot be addressed by a short support response.
- The user explicitly asks to escalate or open a ticket.

Do not escalate minor, non-blocking feedback such as cosmetic UI glitches, typos, wording issues, transient visual oddities, or general observations. For those, acknowledge the feedback, apologize briefly if appropriate, and say it has been noted or can be shared as feedback without claiming escalation.

## Response

Always confirm what action was taken in a friendly, professional tone. Be precise: say "noted" or "shared as feedback" for low-severity issues, and reserve "escalated" for issues that meet the escalation rule.
