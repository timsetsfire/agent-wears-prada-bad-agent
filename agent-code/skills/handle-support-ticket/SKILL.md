# Handling a support ticket

When a user reports an issue or provides feedback, first decide whether the request can be handled directly with standard support guidance or whether it truly requires escalation.

## Escalation rule

Do **not** escalate routine, directly resolvable requests. Resolve directly when the user asks for common support help such as:

- returns or return-policy guidance
- exchanges for size, color, or fit
- wrong item, missing item, or damaged item next steps
- order-status questions
- simple troubleshooting or account guidance

Escalate only when the request requires a human support team action that the agent cannot complete or safely advise on, such as:

- payment, refund, fraud, or billing disputes
- legal, safety, privacy, or account-security concerns
- repeated failures after standard troubleshooting
- requests involving exceptions to policy or manager approval
- missing information that prevents providing useful next steps after asking a concise clarification

## Response

If resolving directly, apologize when appropriate, provide clear next steps, and state that the issue can be handled directly. Do not mention escalation.

If escalating, briefly explain why escalation is needed and confirm what was escalated.

Always use a friendly, professional tone.
