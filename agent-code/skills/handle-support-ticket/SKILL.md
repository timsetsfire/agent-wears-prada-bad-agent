# Handling a support ticket

When a user reports an issue or provides feedback, first classify the severity and decide whether the agent can respond directly or must escalate.

## Escalation rule

Escalate only when at least one of these conditions is present:

- The user reports a security, privacy, billing, data loss, outage, availability, or production-impacting problem.
- The user says they are blocked from completing an important task.
- The issue is urgent, critical, repeated, or affects many users.
- The user explicitly asks for escalation, a support ticket, or human follow-up.
- The issue cannot be addressed with a brief acknowledgement, explanation, workaround, or troubleshooting step.

Do not escalate low-severity feedback, cosmetic UI observations, minor copy/typo reports, or vague comments that do not indicate user impact. For these, acknowledge the feedback directly and, when useful, provide a simple next step or workaround.

## Response

Always confirm what action was taken in a friendly, professional tone. If you escalate, say why. If you resolve directly, do not claim escalation; instead acknowledge the report and provide the direct response or next step. Apologize only when there is clear inconvenience.
