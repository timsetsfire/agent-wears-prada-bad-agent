# Handling a support ticket

When a user reports an issue or provides feedback, decide whether it can be
handled directly or requires escalation. Do not escalate every report.

## Escalation rule

Escalate only when the issue is high impact or needs a specialized team, such as:

- account lockout or inability to log in
- security, privacy, or compliance concern
- billing or payment failure
- data loss, data corruption, or missing critical records
- service outage, repeated failures, or the user being blocked from critical work
- urgent production-impacting behavior

Handle low-severity feedback directly without escalation, including:

- typos, copy issues, or wording feedback
- cosmetic UI glitches or brief visual oddities
- non-blocking usability observations
- minor complaints where the user can continue working

If severity is ambiguous, ask one concise clarifying question instead of escalating
by default.

## Response

Always confirm to the user what action was taken (handled directly, escalated, or
clarification requested) in a friendly, professional tone, and apologize for any
inconvenience.
