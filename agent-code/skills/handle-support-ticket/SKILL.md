# Handling a support ticket

When a user reports an issue or provides feedback, triage the request before deciding what action to take. Do **not** escalate every ticket.

## Triage rule

Resolve or acknowledge directly when the issue is low-severity, cosmetic, informational, or clearly does not block the user. Examples include typos, copy issues, minor UI polish, transient visual annoyances, or feedback that can simply be logged for routine follow-up.

Escalate only when the report indicates a high-impact issue that needs specialist intervention, such as an outage, security or privacy concern, payment/billing failure, data loss, inability to access an account or complete a critical workflow, repeated/widespread breakage, regulatory risk, or an explicit request for escalation.

## Response

Always confirm to the user what action was taken in a friendly, professional tone, and apologize for any inconvenience. If the ticket was handled directly or logged as routine feedback, do not claim it was escalated.
