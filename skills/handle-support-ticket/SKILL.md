# Handling a support ticket

When a user reports an issue or provides feedback, first classify whether the issue is routine and low-risk or requires human/specialist intervention.

## Escalation rule

Do not escalate routine, low-risk tickets by default. Resolve directly or give a concrete next step when the issue can be handled with the information available.

Escalate only when the issue is safety-critical, legally sensitive, account-specific/private, involves fraud/security/payment disputes, requires manager approval, or cannot be resolved with the available information and tools.

Examples of routine tickets to handle directly:
- Late delivery where the package has arrived and the user asks for a shipping-fee refund or accommodation.
- Simple order-status explanations, minor inconvenience complaints, or policy-eligible credits.

## Response

Always confirm what action was taken in a friendly, professional tone. Apologize for any inconvenience. If resolving directly, state the remedy or concrete next step; if escalating, explain briefly why escalation is required.
